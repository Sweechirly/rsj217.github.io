Python-BaseHTTPServer-py-源码分析

发表于 2016-02-17   |   分类于 技术流

BaseHTTPServer 模块封装了HTTP协议的基础方法，提供了HTTPServer和BaseHTTPRequestHandler两个类。HTTP是工作在TCP之上的应用层协议，HTTPServer继承TCPServer类。

HTTPServer

HTTPServer只有一个方法–server_bind。该方法在TCPServer的__init__方法调用，用于进行socket对象的绑定操作。

HTTPServer重写server_bind方法，用于绑定主机地址和监听端口两个字段。

BaseHTTPRequestHandler

BaseHTTPServer模块的主要内容是BaseHTTPRequestHandler，凡是Base开头的类，都会涉及比较多的东西，毕竟是基础嘛。万丈高楼平地起，基础不好，就难说扩展了。

BaseHTTPRequestHandler大体说来有三类方法，第一类用户跟其父类进行接口通信，第二类则是对请求数据读写的方法，第三类就是对数据进行parse和log的方法。

handler

前面介绍BaseReqeustHadnler方法时候，就提到了其handle方法实现了客户端实现处理请求的接口。

handle 方法会判断close_connection字段，通过该字段是否需要重复调用handle_one_request方法，close_connection在handle_one_request会被改变，解析HTTP中header的keep-alive来决定是否需啊哟实现长连接。

handle_one_request

handle调用handle_one_request来处理请求。

该方法首先会从缓存可读文件中读取第一行数据，这一行通常是HTTP协议的第一行，例如GET / HTTP/1.1。如果超过了65537字节(65537是IPv4数据报的大小)，则会设置requestline（请求数据行）， request_version（http版本），command（http方法）三个字段。并发送414错误。其中414表示请求的uri过长。如果没有数据，就修改close_connection为1，并直接返回。结束请求的处理。接着调用父类的finish方法，最后结束tcp连接。如果有数据且没有超过uri的长度设置，就对请求的数据进行解析，调用parse_request方法。parse_request解析之后，再通过自省的方式，判断是否绑定了处理HTTP的具体方法，例如do_GET,do_POST之类，有即调用，没有则会发送一个501错误。一切正常的话，则会更新可写缓冲区。给客户端发送数据。

send_error 和 一些log方法

handle_one_request中，除了解析客户端数据，还有针对错误处理的send_error方法。下面先分析这个方法：

send_error方法会向客户端发送错误信息，并且通过log记录下来。它接受两个参数，一个师错误码，另外一个师错误的消息。因为它会发送一个响应给客户，因此在任何输出内容之前被调用。

send_error通过错误码获取错误信息，在通过_quote_html方法对html实体进行编码，防止CSRF跨站攻击。然后分步调用send_response，send_header，end_headers构造响应的header，最后把响应的错误内容写入缓冲文件中。

send_response

send_response用于构造响应的的header头信，向缓冲区写入response第一行，息。并调用log_request方法， log_request调用log_message方法吧错误信息输入到标准错误输出。最后调用send_header方法。后者将发送MIME 的header信息到缓冲区。

parse_requset

parse_requset 方法对http请求的header进行解析处理。请求数据存储在self.raw_requestline 字段中，解析结果分别使用 command，path，request_version 存储。并通过 MessageClass构造响应的headers。解析构造成功之后返回True，否则发送错误信息给客户端和log到标准错误输出。

parse_request返回之后，handle_one_request会判断结束当前请求还是再进行自省操作。最终返回给客户端，断开连接。
