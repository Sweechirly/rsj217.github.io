项目经验

老虎证券
开户受理系统
  • 技术栈：Python（Tornado）+ Redis + Mongodb
  • 项目简介：老虎证券开户系统是老虎的核心系统之一。开户和交易是券商两大核心业务。我作为公司的研发工程师，主导设计和实现了开户受理系统，并持续开发新功能和维护其正常运转。系统主要使用Tornado构建Web 服务。使用Mongodb作为数据持久化方案，配合Redis做缓存和部分业务需求。
  • 项目成绩：经过多次的优化和改进，从最初的1.0版本到现在开户流程经过了3次大变革，每次都给客服同学提升了近50-70%的工作效率。因此荣获公司月最佳项目奖。
OAuth服务
  • 技术栈：Python（Tornado）+ Redis + Mysql
  • 项目简介：老虎的客户端产品和服务器之间的通信使用了OAuth2协议。作为公司的研发独立设计并实现了OAuth2协议，开发了公司的OAuth鉴权服务。保证了公司客户端和各个后台服务之间的通信安全和便利。该项目依然使用Tornado作为接入网关层，依赖epoll的特性实现高并发。服务上线以来运行高效稳定。
  • 项目成绩：OAuth服务成为了公司产品线上的基础服务，每天作为用户使用产品的入口之一。几次业务核心的变动，OAuth服务很好的配合后端各服务的鉴权需求。代码安全质量经过了安全人员白帽测试。
渠道佣金清算系统
  • 技术栈：Python + Jinja2 + Mysql
  • 项目简介：渠道管理和佣金清算系统也是公司的核心产品之一。该系统提供给公司管理推广渠道和对底层交易通道进行交易清算和佣金计算。使用Python配合Jinja2实现了完整的类CRM系统。
  • 项目成绩：该系统也是由我逐步构建并完善的系统。经过和同事的开发和优化，已经成为了公司不可或缺的一个产品服务。
微服务架构
  • 技术栈：Python+Golang+Nginx
  • 项目简介：公司的产品和服务越来越多。对用户系统需要进行系统拆分。我主导构建了部分服务的拆分职能。即构建以Tornado为基础的前端网关接入层，其后端以Golang实现了微服务。前端和后端采用RPC over HTTP的方式同时，数据格式为jsonrpc2.0。后期也打算支持RPC over TCP的方式。
